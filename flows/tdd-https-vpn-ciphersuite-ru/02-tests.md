# Анализ поведения (Тесты): tdd-https-vpn-ciphersuite-ru

> Версия: 1.0  
> Статус: DRAFT  
> Последнее обновление: 2026-05-11

Этот документ описывает ожидаемое поведение системы через набор тестовых сценариев. Дизайн системы вытекает из этих случаев.

## Сценарии поведения

### 1. Регистрация провайдера
**Описание**: Провайдер "ru" должен быть доступен в глобальном реестре после импорта.

- **Case 1.1: Успешное получение провайдера**
  - **Given**: Пакет `crypto/ru` импортирован.
  - **When**: Вызывается `crypto.Get("ru")`.
  - **Then**: Возвращается объект провайдера и `ok == true`.
  - **Success Criteria**: Провайдер не nil, имя соответствует "ru".

### 2. Конфигурация TLS
**Описание**: Провайдер должен корректно настраивать `tls.Config` для использования алгоритмов ГОСТ.

- **Case 2.1: Установка версии TLS**
  - **Given**: Новый `tls.Config`.
  - **When**: Вызывается `provider.ConfigureTLS(cfg)`.
  - **Then**: `MinVersion` должна быть не ниже TLS 1.2 (согласно RFC 9189).
  
- **Case 2.2: Настройка Cipher Suites**
  - **Given**: `tls.Config`.
  - **When**: Вызывается `provider.ConfigureTLS(cfg)`.
  - **Then**: В списке `CipherSuites` должны присутствовать идентификаторы ГОСТ (например, `0xC100`).

### 3. Криптографические примитивы (ГОСТ)
**Описание**: Алгоритмы должны выдавать корректные результаты согласно тестовым векторам стандартов.

- **Case 3.1: Хеширование Стрибог (256/512)**
  - **Given**: Известная входная строка.
  - **When**: Вычисляется хеш.
  - **Then**: Результат совпадает с контрольным значением из ГОСТ Р 34.11-2012.

- **Case 3.2: Шифрование Кузнечик/Магма**
  - **Given**: Открытый текст и ключ.
  - **When**: Выполняется шифрование в режиме MGM или CTR.
  - **Then**: Зашифрованный текст совпадает с тестовым вектором ГОСТ Р 34.12-2015.

### 4. Обработка сертификатов
**Описание**: Система должна правильно определять тип сертификата для выбора провайдера.

- **Case 4.1: Сквозное тестирование туннеля (Integration)**
  - **Given**: Работающий VPN сервер с провайдером "ru".
  - **When**: Клиент подключается и запрашивает ГОСТ-набор.
  - **Then**: Соединение устанавливается, данные передаются в защищенном виде.
  - **Verification**: `go test -v crypto/ru/integration_test.go`

## Граничные условия и ошибки

- **Case E.1: Неподдерживаемый Cipher Suite**
  - **When**: В конфиге указан несуществующий ГОСТ-набор.
  - **Then**: Система должна выдать ошибку при инициализации.

- **Case E.2: Невалидный ключ ГОСТ**
  - **When**: Попытка загрузки поврежденного приватного ключа ГОСТ.
  - **Then**: Ошибка `failed to load certificate`.
